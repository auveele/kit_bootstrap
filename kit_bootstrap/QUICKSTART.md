# Quickstart

1. Completa los inputs del proyecto en `INIT.md`.
2. Copia las plantillas de `kit_bootstrap/` hacia `docs/`.
3. Rellena placeholders `{{...}}` con datos reales.
4. Define SoT por tema en Document Index.
5. Define roles ultralight + RACI + playbooks por rol.
6. Crea capsulas de informacion clave y context packs para LLM.
7. Ejecuta control de duplicidades y validacion de enlaces.
8. Actualiza release notes con el bootstrap inicial.

Resultado esperado:
- sistema documental operativo,
- roadmap por fases,
- gates y backlog,
- roles claros sin sobreproceso,
- capsulas reutilizables para prompts,
- trazabilidad para futuras sesiones.
